# 🎨 UX Review：删除计时器功能扩展

**文档版本**: v1.0 | **创建日期**: 2026-03-02 | **对应 PRD**: stage1-PRD.md v1.0 | **状态**: 已审查

## 一、Review 范围

对 PRD v1.0 中的以下功能进行 UI/UX 审查：
1. 编辑器被动删除检测（无 UI，纯后台逻辑）
2. 编辑器恢复检测（Undo 恢复，纯后台逻辑）
3. Running 计时器删除时时长结算（纯后台逻辑）
4. Sidebar / StatusBar 即时同步（已有 UI 的状态更新）

**核心特点**：本需求不引入任何新 UI 组件，所有功能为"后台自动处理"。UX Review 聚焦于：
- 已有 UI 的状态同步正确性
- 用户操作的无感体验
- 边界交互（剪切粘贴、Undo/Redo）后的用户感知

## 三、优化建议

### UX-01：Sidebar 卡片删除/恢复过渡动画

**问题**：Sidebar 卡片在被动删除时突然消失，可能让用户困惑。

**建议**：卡片消失时增加 0.2s 淡出动画，恢复时增加淡入动画。

**优先级**：Nice to Have

**决策**：本期暂不实现。理由：
1. 现有 `onTimerRemoved` 直接移除 DOM 节点，与 Sidebar 其他删除行为一致
2. 被动删除发生时用户焦点在编辑器，不太会注意 Sidebar 变化
3. 后续可统一优化所有 Sidebar 动效

### UX-02：恢复后 Notice 提示

**问题**：Ctrl+Z 恢复后计时器变为 paused，Sidebar 重新出现卡片，但用户可能不注意。

**建议**：恢复时发送轻量 Obsidian Notice（如 "Timer restored"）。

**优先级**：Nice to Have

**决策**：本期暂不实现。理由：
1. 恢复是 Ctrl+Z 的自然结果，用户通过编辑器中 span 重现即可确认
2. 频繁 Notice 干扰编辑体验（Undo/Redo 可能快速连续触发）
3. StatusBar 即时更新计数，提供间接反馈

## 四、PRD 修改结论

**无修改**。PRD 功能设计合理，交互方案与 Obsidian 原生行为一致：

- ✅ 被动删除不中断编辑流（无弹窗）
- ✅ 恢复后统一为 paused 状态（不自动 running，避免意外计时）
- ✅ Sidebar / StatusBar 即时同步
- ✅ 幂等性保障（多面板不重复处理）
- ✅ 无新 i18n key 需求

两个 Nice to Have 建议（动画 + Notice）均推迟到后续统一优化。

## 五、总结

本需求是一个**纯后台逻辑增强**，不引入新 UI 元素。现有的 Sidebar/StatusBar 同步机制（`onTimerRemoved` / `onTimerAdded` / `updateStatusBar`）已经能正确处理删除/恢复后的 UI 状态更新。PRD 的交互设计成熟，无需修改。
